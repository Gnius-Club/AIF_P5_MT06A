📝 De un Vistazo: Comando Central

Todo lo que necesitas para preparar y ejecutar esta misión en minutos.

🎯 Objetivo de Aprendizaje

Los estudiantes podrán: Comprender cómo los datos espaciales sirven como base para la toma de decisiones en un robot. Usar el Micro:Bit, el Kit Nezha Pro y la Smart A.I. Lens para realizar lecturas de posición X y programar condicionales de giro.

🤖 Prompt de Referencia para XOLO (Solo para Maestro)

⚠️ NOTA: XOLO ya conoce el tema de esta misión. Este prompt es SOLO de referencia en caso de emergencia técnica: 
Dame una explicación guiada sobre cómo una IA de visión puede entregar más información que solo el color: cantidad de objetos y posición horizontal (X). Analicemos cómo los robots usan estos datos para decidir hacia dónde moverse, y por qué esta lógica es similar a cómo los humanos ajustan su movimiento al “ver” algo a la izquierda o derecha. Ayúdame a reflexionar sobre responsabilidad: una mala interpretación de datos puede provocar errores físicos. Finalmente, guíamecon el diseño conceptual de un robot que gire o se alinee según la posición del color detectado.

⚡ Puntos Esenciales de la Misión

🎓

Concepto Clave

Percepción Espacial: Es la capacidad de un robot para entender dónde están los objetos (izquierda, derecha o centro) usando datos, similar a cómo nosotros usamos nuestros ojos para saber hacia dónde caminar.

🔧

Habilidad Técnica

Usar la cámara inteligente para detectar la posición de un color y hacer que el robot gire hacia él.

🧪

Experimentación

Probar con diferentes posiciones del objeto y distancias para entender cómo cambian los valores de X y las limitaciones del sensor.

💬

Interacción con XOLO

XOLO ya conoce a cada estudiante y el tema de hoy. Solo abren el chat y comienzan a aprender. Tú guías el timing y motivación.

🚨 Resolución Rápida de Problemas

  • Si los alumnos no pueden ingresar a Gnius Space: Activa XOLO SOS en xolo.gnius.club para continuar la misión sin interrumpir la clase.
  • Si el robot no responde: Verificar que la batería del Kit Nezha esté cargada y bien conectada. Revisar que el micro:bit esté correctamente insertado.
  • Si la cámara/lente no reconoce: Revisar la iluminación del salón (evitar luz directa o muy tenue). Pedir a los alumnos acercar o alejar el objeto para mejorar la lectura.

📊 Diapositiva 3: Brain Spark

1. 💬 Pide a XOLO que te ayude a crear:

"Xolo, ayúdame a idear un proyecto aplicado donde el robot use la posición X del color para tomar decisiones. Vibe sugerida: “robot buscador que se alinea con un objetivo”. Se define qué ocurre si el objeto está a la izquierda, centro o derecha. Se diseña la lógica del proyecto y se genera el GCS correspondiente, convirtiéndolo en código Python de acuerdo con el entrenamiento en Nezha, A.I. Lens y micro:bit."

2. 🔬 Experimenta y descubre con tu programa:

  • ¿El robot gira hacia el color como esperabas?
  • ¿Qué pasa si mueves el color muy rápido de un lado a otro?
  • ¿Funciona igual con mucha luz que con poca luz en el salón?
  • Si el robot interpreta mal la posición X, ¿qué consecuencias físicas tendría en un entorno real?
  • ¿Por qué es necesario que el robot reciba datos constantes para mantener su percepción espacial?

¡Es hora de crear! Vamos a convertir a sus robots en buscadores expertos. Imaginen un robot que debe alinearse perfectamente con una señal de rescate. Usaremos el flujo de siempre: primero ideen con XOLO, definan el VIBE de su robot buscador, obtengan su GCS y pásenlo a bloques en MakeCode. Por ejemplo, podrían programar que si el objeto está a la izquierda (X menor a 100), el robot gire a la izquierda hasta centrarlo. Prueben primero en el simulador y luego con su hardware real. No olviden las preguntas de experimentación para ver qué tan preciso es su buscador. Si tienen errores o dudas, pregúntenle a XOLO, él los ayudará a depurar.
